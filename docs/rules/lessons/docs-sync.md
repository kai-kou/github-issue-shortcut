# Warm 層 教訓 — ドキュメント・実装の同期（desync 防止）

ドキュメントと実装（コード・スキル手順）の乖離防止に関するカテゴリ別教訓（タスク依存で Read）。

---

## L-094: ドキュメントと実装が乖離（desync）して「ローカル必須」等の古い前提が残る（2026-06-13）

**パターン**: インフラ・ツールはクラウドで動くように更新されたのに、ドキュメントやスキル手順が
「ローカル実行が必要」「この機能は未対応」と古いまま残り、本来自律実行できる作業をユーザー確認に
回してしまう（CP-6 違反の温床）。逆に、ある処理が SKILL.md 手順で実装されているのかコードで
実装されているのかが曖昧で、二重実装・不整合が起きる。

**根本原因**: ドキュメント（説明）と実装（実体）の更新が同期しておらず、Single Source of
Truth が複数箇所に分散している。

**対策**:
- 「ローカル必須」と書く前に **実際にクラウド実行を試す**（`user-confirmation-minimization.md` §4）
- 実装場所（SKILL.md 手順 / コード）を表に **正確に記載** する（どちらで実装したか明示）
- SSOT を 1 箇所に定め、派生ドキュメントは参照のみにする（独立した重複定義を作らない）
- 機能追加時はドキュメントとコードを同一 PR で更新する
